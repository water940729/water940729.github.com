hexo部署到github笔记

2016-07-31
  • 技术笔记
  • HEXO GitHub

整个环境是是搭在centos6.5上面的，所以我只介绍centos6.5安装过程，亲试。

#第一步：安装node
wget http://nodejs.org/dist/v0.10.26/node-v0.10.26.tar.gz
tar -zvxf node-v0.10.26.tar.gz
./configure --prefix=/usr/local/node
make && make install

#第二步：安装git
这个可以直接yum安装，然后配置git:
git config --global user.email "xx@xx.com"
git config --global user.name "name"
ssh-keygen -t rsa -C "xx@xx.com"

#第三步：安装Hexo
sudo npm install-g hexo
初始化，并指定blog目录
hexo init blog
生成静态页面
hexo generate
开启本地服务
hexo server

#第四步：配置github
建立与你用户名对应的仓库，命名规范 你的用户名.github.io
然后配置_config.yml，注意配置项与值之间的空格，也就是冒号后面要有空格

deploy:

type: git

repo: git@github.com:water940729/water940729.github.io.git

branch:master

这里我用的是ssh的方式，这里如果用https的话需要添加上用户名密码，指的是github上的用户名密码，一种是直接保存在本机，还有一种是直接写到url中
https://yourname:password@git.oschina.net/name/project.git

执行hexo deploy如果出现

Hexo 3.0: ERROR Deployer not found: git
解决办法npm install hexo-deployer-git –save

hexo deploy的时候我也犯了一个很弱智的错误，因为安装在/usr/local，权限是755，所以我普通用户是不能安装的，我一直都是用的sudo进行的上面操作，等到sudo hexo deploy的时候，就一直会报错Permission denied (publickey)，后来我才知道，原来ssh公钥是保存在你普通用户下的，配置在github上的公钥也是你普通账户，所以你用sudo的时候,此时的用户就称为sudoer，所以就会出现denied。解决办法，将你安装目录改为普通用户权限，即让普通用户有权限执行hexo deploy
